09 — Future-Proofing
Hooks for Phase 2 (Backend)

What I'll build in so Phase 2 is plug-and-play:

  • Service layer: All data fetched via Angular services (NoticeService, InstitutionService, etc.) — Phase 2 just swaps mock JSON for HTTP calls.
  • Environment config: environment.ts with apiUrl placeholder.
  • Models/Interfaces: TypeScript interfaces matching planned MongoDB schemas (Institution, Notice, Member, Department, NewsArticle).
  • Routing: Lazy-loaded admin module — route exists but redirects to "coming soon" page now.
  • Form validation: Reactive forms ready for backend submission.
  • Auth guards: Stub AuthGuard in place for admin routes.
  • i18n ready: Structure supports Bangla/English toggle later.
